Esquema de cada expediente

Campo ¿Qué hace? Relación / Tipo de Dato
Socio Vincula el expediente a un socio específico. Al ser único, garantiza que cada socio tenga solamente un expediente. Uno-a-Uno (Socio)
Estatus Registra el estado actual en el que se encuentra el expediente dentro del flujo (ej. Nuevo, Completo, Rechazado). Muchos-a-Uno (Estado)
Usuario Servicios Almacena ejecutivo de servicios encargado de la gestión inicial o creación del registro. Muchos-a-Uno (User)
Usuario Credito Identifica al usuario del área de crédito asignado para revisar, validar o rechazar el checklist del expediente. Muchos-a-Uno (User)
Usuario Archivo Registra al usuario responsable de recibir y resguardar físicamente el expediente en el archivo. Muchos-a-Uno (User)
Fecha Creacion Guarda la fecha exacta en la que se dio de alta el expediente en el sistema de manera automática. Fecha
Usuario Negocios Guarda la referencia del ejecutivo de negocios. Muchos-a-Uno (User)
Fecha Archivado Almacena la fecha en la que el expediente físico fue guardado de manera definitiva en el archivo general. Fecha

Gestión de Citas

Dentro del sistema se pueden generar citas para organizar la entrega física de la documentación. Al programar una cita, es posible asignar uno o varios expedientes simultáneamente.

Requisito

Únicamente se pueden generar citas si el expediente se encuentra en el estado Completo.

Flujo de Confirmación

Al crear la junta (indicando día, hora y expedientes), se enviará un correo automático de confirmación para que el ejecutivo de servicios pueda aceptar o reagendar.
